2) Job of the Linker
-----------------
- Combines object files (.o) into a single executable.
- Resolves references between different files (functions, variables).
- Links program with required libraries (e.g., stdio).
- Produces final executable like 'sum', 'hello', etc.

Practical example:
- In sum.c + sum_func.c, the linker connects sum() function with main.

3) Job of the Loader
-----------------
- Loads the executable into memory (RAM) for execution.
- Sets up memory sections: text, data, bss, stack, heap.
- Initializes program counter at the entry point (main).
- Prepares program to run.

Practical example:
- When executing './hello' or './sum', the loader prepares the program in RAM and starts execution.

4) When and How Linking and Loading Occurred
---------------------------------
- Linking happened during compilation step when 'make' was run:
    gcc src/sum.c src/sum_func.c -o sum
  - Compiler converts .c files to .o files.
  - Linker combines them into executable 'sum'.
- Loading happened during execution:
    ./sum
  - Loader loaded executable into memory and started program execution.
